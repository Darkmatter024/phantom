## 1. Executive Summary

**Overall health:** Core is solid: 96 of 120 verified behaviors work, and the load-bearing pillars (offline-first Master parse with reentrancy guard + SHA cache + chunked yielding, honesty-locked scan verdicts, memoized rollup cache) are real and correctly implemented, not cosmetic. The damage is concentrated in two zones: (1) the Tribal Knowledge pillar's persistence layer — Ghost Echo export AND import are both fully broken by an IndexedDB store-name mismatch, so logged intel can never leave or enter the device, defeating the shift-handoff purpose; (2) a cluster of HUD/phase-pipeline features that were half-rewritten across versions and left with dead CSS (Skyline), inert handlers (deploy_scrollToPhase ignores its argument), stub data (0/0 tasks hardcoded), and a dead voice-button subsystem (vaBtn has no DOM element). Net: the app runs and survives offline, but its knowledge-transfer and several recent UI affordances are illusory.

**Feature verification (120 traced):** ✅ **96 work** · ⚠️ **18 partial** · ❌ **6 broken** — of 120 total.

**Cleanup surface:** 9 confirmed-dead JS symbols (adversarially verified), 27 dead CSS selectors, 10 half-wired/orphaned items.

### The 3 biggest issues
1. Ghost Echo Export AND Import are both broken (data-loss class) — verified at lines 40448-40507: exportGhostEcho/importGhostEcho operate on IndexedDB store 'incidents' while the live GE module writes/reads store 'ghosts' (36985, 36999), and export opens the DB with no version so 'incidents' is never even created. Every echo logged via the sheet is unexportable and any import is invisible to the GE UI. This silently nullifies the TRIBAL KNOWLEDGE pillar (shift-to-shift intel transfer), the entire reason the feature exists.
2. An entire family of HUD/phase features is dead or stubbed and misleads the operator: Rack Skyline never renders (full CSS at 8401-8471 but zero script emits any sky-* class); per-phase task counter is hardcoded to '0/0 tasks' (27609) with no increment path; deploy_scrollToPhase(type) ignores its argument and always scrolls to #deploy-rack-lookup (26458-26461) so every phase node taps to the same place despite an aria-label promising per-phase nav. These present as working tactical UI but convey false/static state.
3. The vaBtn floating push-to-talk subsystem is completely inert — no element with id='vaBtn' exists in markup, createElement, or any template literal (only getElementById reads at 15899-15900, 36369, 36424). vaInit early-returns on the null check so no press-to-talk listeners ever attach and setBtnState never paints listening/thinking visuals. The offline CSS gate also targets the non-existent .va-btn, and the REAL trigger (.qa-voice, 11264) is neither .va-btn nor data-requires-net, so it is not visually gated offline despite the offline-affordance the CSS promises (TACTICAL ASSIST + offline-UX gap).

### The biggest opportunity
Fix the Ghost Echo store-name mismatch — it is the single highest-leverage repair. Aligning exportGhostEcho/importGhostEcho to store 'ghosts' (and opening the DB at the correct version) is a small, surgical change (lines 40450, 40454-40457, 40481-40493) that instantly restores the entire TRIBAL KNOWLEDGE pillar — turning a feature that currently loses all cross-shift intel into a working handoff path — with far less effort than any of the half-rewritten HUD features.

**Highest-leverage addition:** Eliminate the workbook-decode UI freeze in phantom_parseMaster — ship the cheap paint-before-decode mitigation now (await rAF+setTimeout immediately before XLSX.read at 24180 and before each sheet_to_json at 24231/24281, behind a visible 'DECODING MANIFEST…' boot state), and schedule the Web Worker decode as the follow-on. — It is the only feature in the entire verification pass graded below 'works', and it sits on the single most load-bearing path of the whole app: getting the manifest into memory. Every other pillar — Build, Scan, Know, Master, the AI assist — is downstream of a successfully loaded Master. The failure mode is the precise nightmare this app exists to prevent: a gloved operator in a cold aisle, time-boxed, sees the iPad freeze with zero feedback during a multi-second synchronous decode, concludes it crashed, and force-quits mid-parse — destroying the OODA loop and risking a half-written state. Unlike the dead-code excision (cosmetic) or the headroom HUD (preventive), this is an active, reproducible, trust-destroying break on the import path that the data layer otherwise hardens beautifully. The paint-mitigation is S-effort and reversible, so the leverage-to-risk ratio is the best on the board: a few lines convert a silent freeze into a legible 'working…' state for the most critical operation in the product.

## 5. Appendix — Method, Coverage & Limits

**Audit method.** A deterministic multi-agent workflow over the live file: a recon pass (pillar location + function census), 13 parallel feature-cluster verifiers that traced entry→handler→end-state, a dead-JS finder seeded by the census, an **adversarial verification pass** (one skeptic agent per dead-code candidate, instructed to disprove deadness via string-dispatch + template-literal + on*-attribute grep), then grounded recommendations and synthesis. 29 agents total.

**Traced vs assumed.** This synthesis rests on a multi-agent trace of the live 43,441-line file, and I independently re-verified the six most consequential claims by reading the exact regions: Ghost Echo store mismatch (GE STORE='ghosts' at 36985/36999 vs export/import 'incidents' at 40450/40484-40486 — CONFIRMED); deploy_scrollToPhase ignoring its type arg (26458-26461 — CONFIRMED); hardcoded '0/0 tasks' (27609 — CONFIRMED); vaBtn having no DOM source, only getElementById reads (grep 15899/15900/36369/36424 — CONFIRMED); Rack Skyline sky-* being CSS-only with zero JS emission (all 15 grep hits in 8401-8471 stylesheet — CONFIRMED); and the Master-parse sync/async boundary (XLSX.read at 24180-24187 is synchronous and un-chunked — CONFIRMED). The remaining ~108 'works' entries and the finer partial nuances (e.g. memoization generation counters, _metalMatch verdict mapping) are taken from the agents' line-cited traces and were NOT each re-walked by me in this pass — treat them as well-evidenced but not independently re-confirmed here.

**Could not determine:**
- Whether large-payload Master parse actually janks on real-world workbooks: the un-chunked XLSX.read (24180) + two synchronous sheet_to_json calls (24231, 24281) WILL block the main thread, but the actual freeze duration depends on real file sizes/device CPU that cannot be measured by static trace — needs runtime profiling on a representative multi-MB manifest on target iPad hardware.
- Whether any Ghost Echo data currently exists in a 'ghosts' store on real operator devices (i.e. the live blast radius of the export/import break) — IndexedDB contents are device-local and not inspectable from source.
- The full real-world reachability of the legacy Handoff sheet: phantomHandoffOpen (39898) has zero callers in the file, but a runtime check could not rule out injection via console/bookmarklet or a build artifact outside this single file.
- Whether the Port Map proxy (PHANTOM_PROXY_URL) is currently reachable/healthy — only the absence of an offline fallback was confirmed; live proxy status was not probed.

**Coverage gaps (warrant a follow-up pass):**
- The full DEPLOY/master_ surface: only the parse pipeline, rollup memoization, phase-pipeline render, and verdict mapping were traced. deploy_ has 92 defs and master_ has 37 — the majority of these (rack CRUD, audit-walk, floor map, NERVE next-action engine) were sampled, not exhaustively walked.

- Voice Assistant happy path: the dead vaBtn subsystem and the smart-search intent route (which opens the sheet but never calls vaStart) were traced, but the working .qa-voice path (11264 -> openVaSheet + vaStart) and the full SpeechRecognition lifecycle / offline graceful-failure behavior were not end-to-end verified.
- Ghost Echo READ/matcher side: the export/import break is confirmed, but GE.open/showReplay/getSuggestions pattern-matching logic (the in-app surfacing of echoes) was not traced for correctness — only that imported 'incidents' data can never reach it.
- The 18 'partial' and 96 'works' entries were synthesized from agent traces; concurrency/race conditions (e.g. the importGhostEcho pending-counter + tx.oncomplete ordering at 40494-40496, which sets oncomplete inside the last callback and may miss the event), localStorage quota-exhaustion behavior, and cross-feature state interactions were not systematically audited.
