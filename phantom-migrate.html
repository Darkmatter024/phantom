How to use

  Tap Preview to see what the migration WOULD do. No writes happen on preview.

  Review the report. If the merges, handoff count, and orphans look right, tap Commit.

  You'll then be asked to Confirm — that's when writes happen. Backups are written first.

  
Restore (manual rollback)

  If you ever need to undo this migration, run these in Safari's address bar as javascript: URLs or in any console with access to this domain's localStorage:
  

  localStorage.setItem('phantom_deploy_phases_v1', localStorage.getItem('phantom_deploy_phases_v1_pre142'))

  localStorage.setItem('phantom_deploy_racks_v1', localStorage.getItem('phantom_deploy_racks_v1_pre142'))

  localStorage.setItem('phantom_deploy_assign_v1', localStorage.getItem('phantom_deploy_assign_v1_pre142'))

  
Cleanup (after you're satisfied, days later)

  localStorage.removeItem('phantom_deploy_phases_v1_pre142')

  localStorage.removeItem('phantom_deploy_racks_v1_pre142')

  localStorage.removeItem('phantom_deploy_assign_v1_pre142')
