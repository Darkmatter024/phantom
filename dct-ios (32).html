Down Link — Triage Protocol

Work top-down. Confirm each step before advancing. Never skip physical layer.

STEP 01 — PHYSICAL
Inspect the physical layer first
• Cable fully seated? Pull-tab blocking lock?
• Bend radius violations? Fiber crimped?
• Cracked ferrule or dirty end-face?
• Correct polarity on duplex LC (Tx→Rx)?
• MPO: correct key-up / key-down?
STEP 02 — CLEAN END-FACE
Clean fiber before any DOM check
IEC 61300-3-35 pass/fail standard.
• Zone A (0–25μm): ZERO contamination
• Zone B (25–120μm): No scratches >2μm
• Zone C (120–250μm): Minor particulates OK
Dry first, wet/dry if needed. Clean before re-inspect.
Tool: VIAVI FiberChek · Fluke FI-7000 · Thorlabs
STEP 03 — DOM Rx POWER
Pull real-time Rx power
LOW ALARM or worse: dirty fiber, wrong type, bad splice, dead Tx.
HIGH ALARM: attenuator needed or wrong optic class.
show interfaces Ethernet1/1 transceiver show interface eth1/1 transceiver details mlxlink -d /dev/mst/mt4129_pciconf0 -m
STEP 04 — OPTIC COMPATIBILITY
Confirm optic is vendor-qualified
Arista QSFP vendor lock — bypass with service unsupported-transceiver + key.
Check: speed, reach, fiber type (MMF vs SMF), DDM support.
STEP 05 — FEC & ERROR COUNTERS
Degrade vs. hard-down
• Rising FCS/CRC = marginal signal
• Input errors + no FCS = local RX hardware
• Flapping resets = intermittent physical
• FEC uncorrectable > 0 = escalate now
show interfaces Ethernet1/1 counters errors
STEP 06 — LOOPBACK
Isolate local vs. remote fault
Loopback at your end → port healthy, fault is remote.
Loopback at remote → cable and ports good, check config.
test cable-diagnostics tdr interface Ethernet1/1
STEP 07 — SWAP & DOCUMENT
Cable → optic → port. One at a time.
Log: original part, swap part, result, timestamp.
Tag & bag failed optic: port, date, symptoms. Do not discard.
Common Failure Patterns
Link up — dropping packets
MARGINAL
▶

Dirty fiber or marginal optic. Link negotiates but BER elevated.

  • Check Rx power — within sensitivity window?
  • Rising FEC correctable errors over time?
  • Clean both ends of the fiber run
  • Check for micro-bend at cable management points
⚠ Do not leave a high-FEC-correctable link in production without scheduling maintenance. It will fail.
Rx shows –40 dBm or LOS
HARD DOWN
▶
  • –40 dBm = zero light — not just low
  • Check far-end Tx power: is it transmitting?
  • Verify continuity with visible light source (VLS)
  • Check polarity: Tx lane going into Rx?
  • MPO: flipped key? Wrong trunk polarity (A/B/C)?
  • Broken strand: OTDR to locate break distance
🔴 LOS on both ends simultaneously = cable break, not optic failure.
Interface flapping / resets climbing
INTERMITTENT
▶
  • Thermal: Tx power varies with temp — reseat, observe
  • Dirty connector: particulates causing intermittent LOS
  • Loose LC: check clip lock is engaged
  • Cable stress at patch panel entry
  • Bad QSFP/SFP: swap optic first
⚠ 60-second flaps = LACP/keepalive timer. Physical flaps are irregular.
Optic not recognized by switch
COMPAT
▶
  • Arista: look for "Not supported" in transceiver output
  • Check vendor lock — Arista requires key for 3rd-party
  • EEPROM blank: presents as unknown even if approved
  • Try known-good same-type optic to isolate port vs. optic
switch(config)# service unsupported-transceiver <KEY>
switch(config)# interface ethernet 1/1 vendor-override

DAC / AOC / SC Reference
DAC Cable — Direct Attach Copper
PASSIVE / ACTIVE
▶
What Is a DAC?

A DAC (Direct Attach Copper) cable is a fixed-assembly copper twinax cable with transceivers permanently attached at each end. It plugs directly into QSFP, QSFP-DD, or SFP+ ports and does not use fiber. No optic, no DOM light readings — it transmits electrically.

Passive vs Active
TypeReachPowerDOMUse Case
Passive DAC≤3m (typically 1–3m)~0.1WLimited — no Tx/Rx powerToR to spine, NVLink node-to-node
Active DAC3–10m~0.5–1WMore complete DOMLonger rack-to-rack runs
DOM Behavior — Critical Field Note
⚠ Passive DAC cables do NOT report Rx power via DOM. A reading of –40 dBm or "N/A" on a DAC link is NORMAL. Do not condemn the cable based on Rx DOM alone.

Active DACs may report partial DOM. Tx power is usually present. Rx power may show as 0 or absent. Always verify link state via interface counters, not DOM alone.

Speed / Form Factor Matrix
Form FactorSpeedCommon Use
SFP+ DAC10GLegacy ToR, management
QSFP+ DAC40GH100 NVSwitch links, older spine
QSFP28 DAC100GConnectX-7, Arista 7050CX3
QSFP-DD DAC400GGB300 NVLink, Arista 7800
OSFP DAC400G / 800GNext-gen spine, QM9700
Troubleshooting DAC Links
CHECK 01
Verify link state — not DOM
Run show interfaces on Arista or equivalent. Check input/output errors and FEC counters. DAC DOM is unreliable for fault diagnosis.
CHECK 02
Re-seat both ends
DAC connectors can work loose under cable weight. Pull and firmly re-seat both ends. Verify latch is engaged.
CHECK 03
Check bend radius
DAC twinax is stiffer than fiber. Tight bends near the connector cause signal degradation. Minimum bend radius: 4× cable diameter.
CHECK 04
Verify vendor compatibility
DAC cables are subject to the same vendor lock as optics on Arista. Run service unsupported-transceiver if the DAC is not recognized.
🔴 Never use a DAC where reach exceeds its rated distance. Signal integrity degrades rapidly beyond spec. At 400G, even 0.5m over spec can cause uncorrectable FEC errors.
AOC Cable — Active Optical Cable
OPTICAL
▶
What Is an AOC?

An AOC (Active Optical Cable) is a fiber cable with active electrical-to-optical conversion circuitry permanently built into each connector. It looks like a DAC but uses fiber inside. It plugs into the same QSFP/SFP ports as a DAC but transmits optically, giving it much greater reach.

DAC vs AOC — When to Use Which
DACAOC
Reach≤10m1–100m
PowerLowerHigher (~1–2W)
WeightHeavierLighter
Bend radiusStifferMore flexible
DOMLimitedFull Tx/Rx power
Best forSame-rack, ≤3mCross-rack, ≤100m
DOM on AOC

AOCs report full DOM including Tx and Rx power. A healthy AOC typically shows Rx in the –1 to –7 dBm range depending on cable length. Use DOM thresholds from the DOM/DDM tab for alarm reference.

Troubleshooting AOC Links
CHECK 01
Check DOM Rx power
Unlike DAC, AOC Rx power is meaningful. Rx below –14 dBm on a short AOC run is a red flag. Could indicate physical damage to the internal fiber.
CHECK 02
Inspect for kinks
AOC internal fiber is fragile. Any sharp kink or excessive bend can break fibers inside the jacket with no external sign of damage. Check the full cable run.
CHECK 03
AOC is not field-repairable
Unlike discrete optic + fiber, AOC is a single assembly. If the internal fiber breaks, replace the entire cable. No splice, no clean, no fix.
🔴 Do not use AOC in high-temperature environments without verifying the cable's operating temperature spec. Active circuitry in the connector is temperature-sensitive.
SC Connector — Field Reference
SC · DUPLEX
▶
SC Overview

SC (Subscriber Connector) is a push-pull fiber connector with a 2.5mm ferrule. Older than LC but still common in legacy infrastructure, campus runs, and some OOB management links. You will encounter it in inherited environments.

SCLC
Ferrule diameter2.5mm1.25mm
Connection stylePush-pull latchDuplex clip
SizeLargerSmaller — preferred in DC
Common useLegacy, campus, OOBSFP+, QSFP, all modern DC
PolarityBlue = UPC, Green = APCBlue = UPC, Green = APC
APC vs UPC on SC
⚠ NEVER connect SC-APC (green, 8° angled) to SC-UPC (blue, flat). The angled ferrule will not seat correctly and will cause ~60dB insertion loss. The link will not work and you risk scratching both ferrules.
Cleaning SC Connectors

SC ferrules are 2.5mm — use 2.5mm cleaning tools, not 1.25mm LC tools. IEC 61300-3-35 zones apply identically. Zone A (core area) must be contamination-free.

SC to LC Adapter

SC-to-LC hybrid adapters exist for connecting legacy SC runs to modern LC-based equipment. Verify the adapter is rated for the wavelength in use. Insertion loss through a hybrid adapter is typically 0.3–0.5 dB additional.

ℹ If you encounter SC connectors in a modern GPU data center they are almost certainly on legacy OOB management links, console runs, or inherited infrastructure — not on production HPC fabric.

Fiber Certification — OLTS

Optical Loss Test Set (OLTS) certifies fiber attenuation meets TIA-568 / IEC 14763-3 loss budgets. Different tool from the copper certifier.

OLTS Pass/Fail Thresholds
FIBER
▶
Fiber TypeWavelengthMax Loss / kmConnector Budget

What is OOB?

Out-of-Band management is a dedicated network path to server BMCs and switch management ports that is physically separate from the production data network. When in-band is completely dead — OS crashed, NIC failed, network misconfigured — OOB is how you get in without touching hardware.

ℹ OOB should be on a separate switch, separate VLAN, and ideally a separate UPS leg from production. If OOB shares a failure domain with production, it fails when you need it most.

Hot Aisle / Cold Aisle — Field Checks
AIRFLOW
▶

Hot aisle / cold aisle containment separates cool supply air from hot exhaust air. Breakdowns in containment directly raise inlet temperatures even when CRAC/CRAH units are operating normally.

Common Causes of Thermal Elevation
CHECK 01
Missing blanking panels
Any empty rack U without a blanking panel allows hot exhaust to recirculate into the cold aisle. One missing 1U blank can raise local inlet temp by 2–4°C.
CHECK 02
Cable management blocking rear exhaust
Dense cable bundles routed across the rear of servers restrict hot air exhaust. Check that no cables block more than 20% of rear airflow area.
CHECK 03
Containment curtain gaps
Check aisle containment curtains and doors. Gaps allow hot exhaust to bypass containment and enter cold aisle supply.
CHECK 04
Under-floor grille placement (raised floor)
Verify perforated floor tiles are positioned in cold aisles only and not partially under hot aisle equipment.
ℹ Always measure inlet temperature at the front face of the equipment, not at the CRAC unit or room sensor. Room ambient can read 22°C while server inlet is 31°C due to containment failures.
